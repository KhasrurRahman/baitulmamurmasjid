# Baitul Mamur Masjid — প্রজেক্ট নির্দেশনা

## প্রজেক্ট সারাংশ
বাইতুল মামুর মসজিদের (Leystraße 33, 1200 Wien) জন্য বাংলা ভাষার ওয়েবসাইট — মসজিদ পরিচিতি + ফান্ডরাইজিং ট্র্যাকার (ক্রয়ের জন্য টাকা সংগ্রহ) + গ্যালারি + ডকুমেন্টস।

- স্ট্যাক: Next.js (latest, App Router) + TypeScript + Tailwind CSS
- Node ভার্সন: 24 (nvm দিয়ে ম্যানেজ করা — কাজ শুরুর আগে `nvm use 24`)
- ফন্ট: Hind Siliguri (বাংলা)
- চার্ট: Recharts (বার + লাইন)
- ডেটা সোর্স: Google Sheet (publish-to-web CSV), ISR দিয়ে fetch
- মিডিয়া হোস্টিং: Google Drive লিংক (গ্যালারি ছবি + ডকুমেন্ট PDF)
- হোস্টিং পরিকল্পনা: লোকাল ডেভ → পরে Vercel/VPS
- ডিজাইন স্পেক: `docs/superpowers/specs/` ফোল্ডারে
- npm package name `baitul-mamur` (ফোল্ডারের নামে ক্যাপিটাল/স্পেস থাকায় package.json-এ আলাদা নাম দরকার ছিল)
- এই প্রজেক্ট Next.js-এর breaking-changes ভার্সনে — কোড লেখার আগে `AGENTS.md`-এর নির্দেশ অনুযায়ী `node_modules/next/dist/docs/` চেক করা উচিত

## পেজ স্ট্রাকচার
1. হোম (`/`) — হিরো, ফান্ড প্রোগ্রেস উইজেট, ডিপোজিট কার্ড (QR সহ), সাম্প্রতিক দান প্রিভিউ, গ্যালারি প্রিভিউ
2. মসজিদ সম্পর্কে (`/about`)
3. ফান্ড ডিটেইলস (`/fund`) — সম্পূর্ণ দান হিস্টোরি + চার্ট
4. গ্যালারি (`/gallery`)
5. ডকুমেন্টস (`/documents`) — in-page PDF প্রিভিউ

## ডেটা মডেল (Google Sheet ট্যাব)
- `donations`: date, donor, amount, method
- `gallery`: drive_link, caption
- `documents`: drive_link, title, description
- সাইট কনফিগ (target amount, ব্যাংক/বিকাশ/নগদ নম্বর, QR ছবি) → `lib/site-config.ts`

## এজেন্ট নির্দেশনা — মেমরি (গুরুত্বপূর্ণ)
এই প্রজেক্টে কাজ করা প্রতিটি সেশনে, **ব্যাখ্যা না করে / অনুমতি না চেয়ে**, কাজ শেষে বা বড় কোনো পরিবর্তনের পর `MEMORY.md` ফাইল নিজে থেকেই আপডেট করতে হবে:
- কী কাজ হয়েছে (ফিচার/ফাইল/পেজ)
- কী বাকি আছে
- কোনো সিদ্ধান্ত/কনস্ট্রেন্ট যা ভবিষ্যতে জানা জরুরি (যেমন Sheet ট্যাবের নাম পরিবর্তন, কোনো API/env key)

এই আপডেট নীরবে করতে হবে — ইউজারকে "আমি মেমরি আপডেট করছি" বলে জানানোর প্রয়োজন নেই। শুধু কাজ শেষে ফাইলটা current state অনুযায়ী রিরাইট/আপডেট করে দিতে হবে।

প্রতিটি সেশনের শুরুতে `MEMORY.md` পড়ে নিতে হবে যাতে আগের অবস্থা জানা থাকে।
